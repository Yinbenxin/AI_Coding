> **摘要：** 读写文件前先判断大小和结构性，选择对应策略。

---

## 决策树总览

```
是否是小文件？（写 ≤100 行 / 读 ≤200 行）
├── 是 → 直接读/写
└── 否（大文件）
    ├── 是否是结构性文件（有明确章节/标题层级）？
    │   ├── 否 → 分段读/写
    │   └── 是 → 多步读/写（见下）
```

---

## 写入策略

**小文件（≤100 行）：** 直接 Write 一次写入。

**大文件·非结构性：** 按内容块多次 Write/Edit 追加写入。

**大文件·结构性：** 四步执行：
1. Write 写入文件头部（frontmatter、标题、整体说明）
2. Edit 插入目录及每章节一行摘要
3. Edit 插入所有章节标题占位符（骨架）
4. 按顺序 Edit 逐章节填充具体内容

---

## 读取策略

**小文件（≤200 行）：** 直接 Read 一次读取。

**大文件·非结构性：** 按 offset+limit 逐块读取所需范围。

**大文件·结构性：** 三步执行：
1. Grep 搜索标题关键词或 Read 前 50 行，获取章节列表及行号
2. 根据结构确定目标章节行号范围
3. Read 的 offset+limit 精准读取目标章节；找不到目标时再全量读取

---

## Why

- 写入：单次 Write 内容过长时参数容易截断，导致 file_path 和 content 丢失连续报错
- 读取：大文件全量读取浪费 context，且容易超出限制
